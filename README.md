## ğŸ¤ How to Contribute

### Step 1: Participate in a Contest
Join contests on [CodeChef](https://www.codechef.com/) or [LeetCode](https://leetcode.com/).

### Step 2: Prepare Your Solution
After solving problems, organize your solution files according to the repository structure.

### Step 3: Create Your Folder Structure
Navigate to the appropriate platform folder and create a new folder for the contest:

```bash
# For CodeChef
CodeChef/<Contest_Name>/

# For LeetCode
Leetcode/<Contest_Name>/
```

### Step 4: Add Your Solutions
Place your solution files inside the contest folder with clear, descriptive filenames.

### Step 5: Commit Your Changes
Follow this commit format:

```bash
git add .
git commit -m "Add solutions for <Platform> <Contest Name>"
```

**Examples:**
- `git commit -m "Add solutions for CodeChef STARTERS_110"`
- `git commit -m "Add solutions for Leetcode Weekly Contest 372"`

### Step 6: Create a Pull Request
1. Push your changes to your branch
2. Create a Pull Request with a clear title and description
3. Wait for review from the Tech Head

## âœ… Contribution Guidelines

1. **File Naming**: Use clear, descriptive names for your solution files
   - For CodeChef: Use problem codes (e.g., `PARITY.cpp`, `MAKEPAL.py`)
   - For LeetCode: Use problem names in kebab-case (e.g., `two-sum.cpp`)

2. **Code Quality**: 
   - Write clean, readable code
   - Add comments for complex logic
   - Follow language-specific best practices

3. **Contest Folder Naming**:
   - CodeChef: Use contest names like `STARTERS_110`, `LUNCHTIME_105`
   - LeetCode: Use format like `Weekly_Contest_372`, `Biweekly_Contest_120`

4. **Optional README**: Add a `README.md` in your contest folder with:
   - Contest date and link
   - Problem descriptions
   - Your approach/explanations
   - Time and space complexity

5. **One PR per Contest**: Submit all problems from a single contest in one PR
